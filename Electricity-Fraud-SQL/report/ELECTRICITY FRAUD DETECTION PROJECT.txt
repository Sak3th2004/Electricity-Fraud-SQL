ELECTRICITY FRAUD DETECTION PROJECT REPORT

PROJECT OVERVIEW:
Built a SQL system to detect electricity consumption fraud using MySQL database.

DATABASE STRUCTURE:
- 5 tables: customers, consumption, fraud_flags, locality_average, fraud_risk_dashboard
- 60 consumption records from 20 customers
- 12 months of data (Jan 2024 - Dec 2024)

FRAUD DETECTION RESULTS:
Total fraud cases detected: 14
Critical risk customers: 2 (Tamper Singh, Suspicious Co Ltd)
High risk customers: 1 (Constant Consumer)
Detection rate: 23.33% of all readings flagged as critical

KEY FRAUD PATTERNS FOUND:
1. Negative consumption readings (impossible - indicates meter tampering)
2. Usage spikes over 4x normal consumption
3. Identical monthly usage (suggests system bypass)

SQL TECHNIQUES USED:
- Window functions (LAG, LEAD, AVG OVER)
- Moving averages for trend analysis
- Stored procedures for automated detection
- Views for easy reporting
- CTEs for complex calculations

Database: MySQL 9.2.0

- Score 90: Consumption over 5x city average
